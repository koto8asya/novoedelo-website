# BPMN AS IS — Текущие бизнес-процессы ООО «Новое дело»

## Процесс 1: Приём заявки на ремонт

**Участники (swimlanes):** Клиент, Администратор, Мастер, Склад

**Шаги:**
1. Клиент обращается по телефону или приходит лично
2. Администратор записывает данные вручную в Excel
3. Администратор распечатывает бумажную квитанцию
4. Устройство помещается на склад
5. Мастер получает устройство и проводит диагностику
6. Администратор звонит клиенту с результатами
7. При согласии клиента — ремонт выполняется
8. Клиент приходит лично, предъявляет квитанцию, платит, получает устройство

**Проблемы:**
- Время оформления: 10–15 минут
- Высокий риск потери данных
- Нет автоматических уведомлений
- Перегрузка администратора

## Процесс 2: Выполнение ремонта

- Мастер получает устройство со склада
- Проводит диагностику
- Устно сообщает администратору стоимость
- Администратор вручную уведомляет клиента
- После согласия выполняет ремонт
- Обновляет запись в Excel

## Процесс 3: Выдача устройства

- Клиент приходит с бумажной квитанцией
- Администратор ищет запись в Excel
- Оплата наличными или терминалом
- Запись закрывается вручную

> Файл диаграммы: `as-is.bpmn` (Camunda Modeler, BPMN 2.0)
