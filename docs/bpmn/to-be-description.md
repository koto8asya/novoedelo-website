# BPMN TO BE — Оптимизированные бизнес-процессы с веб-ресурсом

## Процесс 1: Приём заявки (оптимизированный)

**Участники:** Клиент, Веб-система, Администратор

**Шаги:**
1. Клиент заполняет онлайн-форму на сайте
2. Веб-система автоматически регистрирует заявку в БД
3. Клиент получает Email-подтверждение с номером заявки
4. Администратор получает уведомление в личном кабинете
5. При визите клиента — электронная QR-квитанция формируется автоматически

**Результат:** Время оформления — 2–3 минуты

## Процесс 2: Выполнение ремонта (оптимизированный)

1. Мастер получает задание через панель управления
2. Вносит результаты диагностики в систему
3. Система автоматически уведомляет клиента (SMS/Email)
4. При согласии клиента — заявка переходит в статус «В работе»
5. Клиент отслеживает статус в реальном времени

## Процесс 3: Выдача (оптимизированный)

1. При готовности — система автоматически уведомляет клиента
2. Администратор сканирует QR-код
3. Система выводит сумму к оплате
4. После оплаты — запись закрывается автоматически, email-чек отправляется клиенту

## Сравнение AS IS vs TO BE

| Показатель | AS IS | TO BE |
|---|---|---|
| Время оформления | 10–15 мин. | 2–3 мин. |
| Уведомления | Вручную по телефону | Авто SMS/Email |
| Риск потери данных | Высокий (Excel) | Минимальный (БД) |
| Онлайн-запись | Отсутствует | Реализована |
| Нагрузка на администратора | Высокая | Снижена на 60% |
